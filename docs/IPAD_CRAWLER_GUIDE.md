# iPad 端爬虫运行指南

## 📖 概述

本指南详细说明如何在 iPad 上运行本地爬虫，实现电量数据采集与上传到云端。

**为什么电脑端不需要配置？**
- 电脑端爬虫和后端服务器在同一个进程中运行，共享 `.env.local` 配置
- iPad 端是独立的 Python 脚本，通过 API 上报数据

**现在的方案（v5.1）：**
- ✅ 通过 API 上报数据，无需直接连接 MongoDB
- ✅ 零配置，一键启动
- ✅ 本地缓存补发（网络恢复后自动上传）
- ✅ 支持 iOS 快捷指令定时触发

---

## ⚠️ iOS 后台限制说明

**重要**：iOS 对后台应用有严格限制，当屏幕关闭或应用切换到后台时，系统会冻结进程。这是 iOS 的安全和电量管理机制，无法通过代码绕过。

**可行的解决方案**：
1. **屏幕常亮**（最简单）：设置屏幕永不自动锁定，连接充电器
2. **iOS 快捷指令**（推荐）：创建定时自动化，每15分钟自动触发一次爬取
3. **服务器部署**（最可靠）：在远程服务器上运行爬虫，不受 iOS 限制

---

## ✅ 前置条件

1. **iPad 设备**：运行 iOS 15 或更高版本
2. **网络环境**：确保 iPad 能访问互联网（建议 Wi-Fi）
3. **充电器**：后台运行会消耗电量，建议连接充电器

## 🚀 方案一：使用 iSH（推荐，无需越狱）

iSH 是一款在 iOS 上运行的 Alpine Linux 模拟器，无需越狱即可使用。

### 步骤 1：安装 iSH

1. 打开 **App Store**
2. 搜索 **iSH Shell** 并下载安装
3. 打开 iSH，等待初始化完成（首次打开需要几分钟）

### 步骤 2：安装依赖

在 iSH 终端中输入以下命令：

```bash
apk update
apk add python3 curl git
```

### 步骤 3：克隆项目并运行

```bash
git clone https://github.com/mariohuang233/hmlcrawl.git
cd hmlcrawl
python3 scripts/mobile_crawler.py --daemon
```

**完成！** 就这么简单，不需要任何配置！

### 步骤 4：保持运行（关键）

#### 方法 1：设置屏幕常亮（推荐）

1. 打开 iPad **设置**
2. 进入 **显示与亮度** → **自动锁定**
3. 设置为 **永不**
4. 连接充电器，保持屏幕常亮

#### 方法 2：使用音乐播放器保活

1. 在 iSH 中运行爬虫
2. 打开音乐 App，播放一首歌曲
3. 按 Home 键返回主屏幕（音乐继续播放）
4. iOS 会认为应用正在播放音频，允许后台运行

### 步骤 5：重启爬虫

如果需要重启爬虫（例如更新代码后）：

```bash
# 按 Ctrl+C 停止当前爬虫
# 然后进入项目目录
cd hmlcrawl

# 更新代码（可选）
git pull

# 重新启动
python3 scripts/mobile_crawler.py --daemon
```

## 🤖 方案二：使用 iOS 快捷指令（推荐，息屏可用）

**这是真正支持息屏后台运行的方案！** 通过 iOS 内置的快捷指令自动化，每15分钟自动触发一次爬取。

### 步骤 1：创建快捷指令

1. 打开 **快捷指令** App
2. 点击右下角 **+** 创建新快捷指令
3. 点击 **添加操作**

### 步骤 2：添加操作

搜索并添加以下操作：

1. **运行 Shell 脚本**（搜索"Shell"）
   - 设置脚本类型为 **bash**
   - 输入以下脚本：
     ```bash
     cd ~/hmlcrawl && python3 scripts/mobile_crawler.py --once
     ```

2. **等待**（可选，搜索"等待"）
   - 设置等待时间为 10 秒（确保爬虫完成）

### 步骤 3：创建自动化

1. 切换到 **自动化** 标签页
2. 点击 **创建个人自动化**
3. 选择 **时间**
4. 设置时间为 **每天**，间隔为 **每15分钟**
5. 点击 **下一步**
6. 点击 **添加操作** → 选择刚才创建的快捷指令
7. 关闭 **询问** 开关（允许自动运行）
8. 点击 **完成**

### 步骤 4：修改爬虫支持单次运行

爬虫已支持 `--once` 参数，会执行一次爬取后退出。

## 🌐 方案三：使用 Termius（SSH 连接服务器）

如果已有 Linux 服务器，推荐通过 Termius 远程连接运行爬虫。**服务器端不受 iOS 后台限制**。

### 步骤 1：安装 Termius

1. 在 App Store 搜索并下载 **Termius**
2. 配置 SSH 连接到你的服务器

### 步骤 2：在服务器上部署

登录服务器后执行：

```bash
# 安装依赖
apt update && apt install python3 git screen python3-pip

# 克隆项目并运行
git clone https://github.com/mariohuang233/hmlcrawl.git
cd hmlcrawl

# 使用 screen 后台运行
screen -S crawler
python3 scripts/mobile_crawler.py --daemon

# 按 Ctrl+A 然后按 D 退出 screen（爬虫继续运行）
```

### 后台运行强化

| 功能 | 说明 |
|------|------|
| **心跳监控** | 每秒检查活动状态，检测是否被系统挂起 |

### 数据可靠性

| 功能 | 说明 |
|------|------|
| **本地缓存** | API 上传失败时自动保存到本地 |
| **自动补发** | 启动时自动上传之前缓存的数据 |
| **数据校验** | 每条记录包含 checksum，确保数据完整性 |
| **重复检测** | 基于 crawl_id 自动去重 |

默认情况下，所有配置已经内置，无需修改。如果你需要自定义，可以编辑 `scripts/mobile_crawler.py`：

```python
# ============ 配置 ============

```

## 🔍 故障排除

### 常见问题

**Q: 运行后显示"未安装模块"**

A: 确保安装了依赖：

```bash
apk add python3 curl git
```

**Q: API 上传失败**

A: 检查网络连接，确保能访问后端地址。尝试切换 Wi-Fi 或使用蜂窝数据。

**Q: 爬取失败，显示"请求失败"**

A: 检查网络连接，确保可以访问目标网站。尝试切换 Wi-Fi 或使用蜂窝数据。

**Q: 解析失败，显示"所有解析策略均失败"**

A: 目标网站可能更新了页面结构。联系开发者更新解析规则。

**Q: 解析出的电量数字明显错误（如 7 kWh）**

A: 这是因为解析策略匹配到了页面上的其他数字。请确保爬取的页面是正确的电表页面，并且包含"剩余电量"字样。

**Q: 息屏后爬虫停止运行**

A: 这是 iOS 的后台限制。请参考以下方案：
- **方案二（推荐）**：使用 iOS 快捷指令定时触发
- **方案一**：设置屏幕常亮
- **方案三**：部署到远程服务器

**Q: 网络恢复后数据是否会自动补发？**

A: 是的！v5.1 新增了本地缓存和自动补发功能，下次启动时会自动上传之前缓存的数据。

## 🔄 与电脑端爬虫的协同

iPad 端与电脑端爬虫共享同一个后端：
- 相同的数据格式
- 相同的爬取间隔（15分钟）
- 相同的解析策略
- 自动去重（基于 crawl_id）

当两端同时运行时，数据会自动合并，确保数据完整性。后端会自动处理重复数据。
